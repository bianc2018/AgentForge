# 访谈指南 — 测试策略

## 原则

1. **推导先于提问。** 产物涵盖了大部分内容：
   - `design.md` 的领域方法 → 单元测试
   - `design.md` 的 repositories 和 adapters → 集成测试
   - `.feature` 的每个 Scenario → 一个 Gherkin 测试
   - 每个可测量的 NFR → 一个性能测试
   - 每个安全 NFR + PRD 的风险 → 安全测试

2. **一次一个问题。**

3. **接受"N/A"。** 并非每个 feature 都有性能 NFR 或安全风险。

---

## 各类型的问题库

### 单元测试

当产物中领域行为不清楚时使用。

- "方法 `<方法>` 是否有 REQ 覆盖之外的边缘情况？（如：空值、空列表、溢出）"
- "Story 的验收标准中是否有未在 REQ 中明确的隐含业务规则？"

### 集成测试

当真实依赖 vs mock 依赖的策略不清楚时使用。

- "对于 `<AdapterX>` 的测试，我们使用沙箱模式的真实外部服务还是本地 mock？"
- "CI 环境是否有 Redis 和数据库可用，还是我们需要内存替代方案？"
- "如果外部邮件/短信服务不可用，测试中应该发生什么？"

### E2E Gherkin

仅在状态操作机制不清楚时使用。

- "为了在测试中强制 OTP 验证码过期，我们是直接操作数据库还是有测试辅助工具？"
- "Scenarios 之间共享的 step definitions 是放在全局文件还是按 feature 分开？"

### 性能

当 NFR 没有指定足够具体的阈值时使用。

- "NFR 说 `<模糊标准>`。可接受的数值是多少？（如：p95、p99、平均值）"
- "性能测试是在每个 PR 的 CI 中运行还是仅定期运行？"

### 安全

用于 PRD 中有技术缓解措施但没有对应 NFR 的风险。

- "PRD 提到了 `<风险>` 的风险。是否计划了自动化测试来验证缓解措施，还是手动验证？"
- "中性响应的时序攻击测试（NFR-3）只检查响应内容还是也检查响应时间？"

---

## 什么不要问

- BDD Scenarios 中已明确的内容 — 每个 Scenario 是一个 GH 测试，无需确认
- NFR 已用数字指定的内容（阈值、TTL、百分位）
- 哪些组件有单元测试 — 所有领域方法都有，无例外
