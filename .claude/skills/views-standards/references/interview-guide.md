# 访谈指南 — 屏幕/视图提取

## 提取原则

1. **推导先于提问。** 组件、字段和错误消息可以直接从
   Scenarios 中提取 — 只问产物没有回答的内容。

2. **一次一个问题。** 如果两个项目模糊，选择最关键的。

3. **开放式问题，非确认性问题。** 优先使用"此屏幕可能的状态有哪些？"
   而非"此屏幕有加载状态吗？"。

4. **错误消息是原文。** 绝不转述 `Then` — 精确复制。

5. **视觉参考始终为空。** 绝不询问线框图或 mockup
   — 保留默认占位符。

6. **其他 feature 的屏幕是外部参考。** 不记录属于其他
   feature 的屏幕 — 仅记录名称/URL 作为导航参考。

---

## 何时接受 vs. 深入挖掘

**接受**当：
- `Then` 中提到的所有表单字段都在组件表中
- 错误消息是 `.feature` 中 `Then` 的原文副本
- 屏幕目的是面向用户的（不描述实现）
- 视觉参考 section 仅包含 3 个默认占位符

**深入挖掘**当：
- Scenarios 中有提到的字段但未出现在组件中
- 错误消息被转述而非逐字复制
- 目的描述实现（"保存到数据库"、"向 /api/ 发送 POST"）
- 加载状态缺失但 Scenarios 描述了异步操作
- 考量 section 为空但 Scenarios 有带验证的 `Scenario Outline`

## 如何从 Gherkin 推导组件

### `Given` → 屏幕名称
```gherkin
Given 访问者在注册页面
```
→ 屏幕：**注册页面** (`pagina-de-cadastro`)

### `When` → 按钮和操作
```gherkin
When 访问者用有效数据填写所有必填字段并提交表单
```
→ 按钮：**提交**（类型：submit，行为：提交注册表单）

### 带有 `包含以下字段的表单` 的 `Then` → 字段表
```gherkin
Then 系统显示一个包含姓名、邮箱、密码、确认密码、出生日期和头像字段的表单
```
→ 字段：姓名（text）、邮箱（email）、密码（password）、确认密码（password）、出生日期（date）、头像（file）

### `Then ... 被带到` / `Then ... 看到页面` → 导航链接 / 成功状态
```gherkin
Then 访问者被带到注册页面
```
→ 当前屏幕上的导航链接；或其他屏幕的成功状态

### Scenario Outline + Examples → 错误表（错误状态）
```gherkin
| 邮箱已关联到现有账户 | 此邮箱已注册。请尝试登录或使用其他地址。 |
```
→ 原文错误消息：`"此邮箱已注册。请尝试登录或使用其他地址。"`

## 各 section 的问题库

### 概述

当目的或 URL 在产物中不清楚时使用。

- "从用户的角度来看，此屏幕的主要目的是什么？"
- "此屏幕有定义的 URL 吗？它在 PRD 或 Scenarios 中出现吗？"
- "名称 '<推导的名称>' 是此屏幕在产品中的已知名称，还是有其他名称？"

### 表单字段

当 Scenarios 中提到的字段需要更多细节时使用。

- "字段 '<字段>' 接受什么类型的输入？自由文本、选项选择、文件上传？"
- "是否有字段在屏幕加载时预填了默认值？"
- "是否有条件显示的字段（仅在另一个字段被填写时）？"

### 按钮和操作

当 `When` 描述了操作但没有明确按钮标签时使用。

- "提交表单的按钮在界面中是如何标注的？"
- "除了提交表单，此屏幕上还有其他操作按钮吗？（如：取消、清除）"
- "按钮在提交期间会发生什么 — 被禁用、显示 spinner？"

### 加载状态

当 Scenarios 描述了异步操作但没有详细说明反馈时使用。

- "当系统处理表单提交时，用户看到什么？"
- "提交按钮在处理期间被禁用吗？"

### 错误状态

当错误消息不完整或需要显示上下文时使用。

- "错误消息 '<消息>' 显示在哪里 — 字段下方还是表单顶部？"
- "字段错误是逐个字段显示还是作为单一列表显示？"
- "此屏幕是否有网络错误或超时的消息？"

### 成功状态

当正常路径的 `Then` 不够详细时使用。

- "成功提交后，用户是留在此屏幕还是被重定向？"
- "成功后是否有确认消息或视觉反馈？"

### UX 考量

当 PRD 或需求提到 Scenarios 未覆盖的验证规则时使用。

- "字段 '<字段>' 是否有 Scenarios 中未出现的格式规则？（如：掩码、最大长度）"
- "是否有字段有特定的输入提示（placeholder）？"
- "此屏幕需要在移动设备上工作吗？是否有预期的布局适配？"

---

## 提取节奏

- 始终宣布正在处理哪个屏幕：`[屏幕 X/N：<名称>]`
- 在提问之前展示完整的 `tela.md` 草稿
- 纳入回答后，展示更新的草稿
- 当屏幕准备好时：`✅ 屏幕 X 已完成：docs/features/<slug>/views/<名称>/tela.md`
- 无阻力地接受跳过或删除屏幕的请求；调整隐式编号

---

## 警告信号 — 需要深入挖掘的草稿

| 观察到的问题 | 如何深入挖掘 |
|---|---|
| 错误消息被转述 | 精确复制 `.feature` 中 `Then` 的文本 |
| 表单字段缺失 | 检查 `Then ... 显示一个包含以下字段的表单` |
| 目的描述实现 | "用户如何描述他在这个屏幕上做什么？" |
| 有提交操作的屏幕缺少加载状态 | "用户在等待响应时看到什么？" |
| 视觉参考 section 被填写 | 删除内容并替换为 3 个默认占位符 |
| 带有技术名称的按钮（"handleSubmit"） | 使用 UI 中显示的标签 — 从 Scenarios 推导或询问 |
| 没有明确目标的导航链接 | 从 `Then ... 被带到` 推导或询问目标 URL |
