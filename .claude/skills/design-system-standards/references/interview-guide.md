# 访谈指南 — Design System

## 原则

1. **提取先于提问。** 项目配置是首要来源 — 从找到的任何文件中提取（CSS、design tokens、UI 库配置）。如果已经存在，不要问。
2. **一次一个问题。**
3. **问题是关于未做出的决策** — 而非已经配置的内容。

---

## 各文件的问题库

### colors.md

当颜色未在找到的配置中时使用。

- "项目有主要的品牌颜色吗？可以是 hex、HSL、RGB 或颜色名称。"
- "是否有用于错误/破坏性状态的颜色，还是我们使用红色作为默认语义？"
- "项目会有暗色模式吗？这影响我们将颜色定义为 CSS 变量还是固定值。"

### typography.md

当项目未配置字体时使用。

- "这个项目有特定的字体还是使用系统默认的 sans-serif？"
- "项目是否有 'display' 排版层级（大号 hero 标题）还是从 H1 开始？"

### spacing.md

当项目有非显而易见的自定义间距时使用。

- "项目使用自定义间距层级还是样式库的默认层级？"
- "页面容器的默认 padding 是多少？（如：移动端较小，桌面端较大）"

### components.md

用于无法从代码推导的组件使用决策。

- "何时使用较低视觉权重的按钮 vs 幽灵按钮（ghost）？项目中有规则吗？"
- "项目有可复用的空状态（EmptyState）组件还是每个 feature 创建自己的？"
- "除了 PageHeader 和 EmptyState 之外，是否有其他出现在多个屏幕上的组合组件？"
- "默认的加载行为是什么 — skeleton（保留布局）还是 spinner（替换内容）？"

### themes.md

当代码中主题策略不清楚时使用。

- "项目从一开始就支持暗色模式还是 MVP 之后？"
- "主题切换是手动的（UI 中的按钮）还是跟随操作系统偏好？"

---

## 什么不要问

- 任何找到的配置文件中的值 — 直接提取
- 任何找到的 CSS 文件中的 CSS 变量 — 直接提取
- 已安装的组件 — 在步骤 0 中检测到的组件目录中使用 Glob 列出
- 已安装的内容 — 通过扫描找到的目录和配置文件推导
