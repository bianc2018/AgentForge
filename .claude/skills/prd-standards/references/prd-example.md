# PRD — 配送员登录

## 概述

允许先前注册的配送员使用其凭据进行认证，从而访问配送平台，
并在系统中启动一个有效的会话。

认证成功后，配送员将能够访问平台的受限功能，
而无需在每次请求时重新认证。

---

## 问题

目前已注册的配送员在平台上没有正式的认证机制。

没有认证，受限功能无法得到适当保护，
系统也无法确保只有授权用户才能执行某些操作。

此外，缺乏持久会话损害了配送员在持续使用系统期间的体验。

---

## 目标用户

通过 `register-user` feature 预先创建了账户的配送员。

---

## 目标

1. 允许配送员使用其标识符和密码进行认证。
2. 在验证凭据后创建有效的认证会话。
3. 允许系统自动识别所提供标识符的类型。
4. 拒绝使用无效凭据的认证尝试。
5. 在正常使用应用程序期间保持用户会话活跃。

---

## 成功标准

| 标准 | 度量 |
| ------------------------------------------------------- | ------------------------------------------------ |
| 配送员在有效登录后能访问系统 | 100% 的有效登录产生活跃会话 |
| 无效凭据被拒绝 | 系统返回认证错误 |
| 会话允许访问受保护功能 | 已认证用户可访问仪表盘 |
| 会话在正常使用期间保持活跃 | 用户无需重复认证 |
| 系统处理滥用的登录尝试 | 保护机制被激活 |

---

## 不在范围内

- 密码恢复
- 邮箱或电话验证
- 社交登录
- 显式登出
- 多因素认证
- 同时多会话控制

---

## 主流程

```mermaid
flowchart LR
    A([配送员访问<br/>登录页面]) --> B[填写<br/>标识符和密码]
    B --> C[系统验证<br/>凭据]
    C --> D[系统认证<br/>配送员]
    D --> E[用户会话<br/>已启动]
    E --> F([配送员被重定向<br/>到仪表盘])
```

---

## 替代流程

### 无效凭据

```mermaid
flowchart LR
    A([配送员提交<br/>错误凭据]) --> B[系统拒绝<br/>认证]
    B --> C[显示错误<br/>消息]
    C --> D([配送员可以<br/>重试])
```

---

## 依赖

- 负责创建配送员账户的 `register-user` feature
- 前端的登录界面

---

## 风险

| 风险 | 缓解 |
| ------------------------------------------------------- | -------------------------------------------------------- |
| 使用无效凭据反复尝试登录 | 应用防滥用保护机制 |
| 未授权用户试图访问系统 | 在访问受保护资源之前要求认证 |
| 无效会话允许不当访问 | 每次受保护访问时验证会话 |
